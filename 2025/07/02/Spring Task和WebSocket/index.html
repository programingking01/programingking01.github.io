WebSocket

1.介绍

  • 双向通信，全双工，长连接
  • 与HTTP的关系：
    • 基于HTTP基础之上的，都是TCP
    • WebSOcket的连接需要通过一个握手（HTTP Upgrade请求），表明切换协议，服务器如果支持WebSocket，会返回一个HTTP 101 状态码，表示协议切换成功
    • 握手完成后，HTTP协议的作用结束，通信会切换为WebSocket协议，双方可以开始全双工通信
  • 数据帧小，传输高效，无需像HTTP每次都携带完整的请求头

![image-20250419173835335](Spring Task和WebSocket.assets/image-20250419173835335.png)

![image-20250419174205653](Spring Task和WebSocket.assets/image-20250419174205653.png)

  • 用于实时场景

  • 和HTTP请求一样，前端和WebSocket服务器之间传输消息时，也可以通过JSON格式对数据进行序列化传输
