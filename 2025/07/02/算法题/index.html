2.力扣76.最小覆盖子串（滑动窗口+哈希表）

  • 思路

  • 统计字符需要出现次数：遍历t串并且，存入哈希表（need）键为字符，值为需要出现次数，同时使用一个变量needCount记录需要匹配的不同字符的数量

  • 初始化窗口：使用两个指针 left 和 right 表示窗口的左右边界，初始时都指向字符串 s 的第一个字符。另外，使用一个哈希表 （window） 记录当前窗口中每个字符的出现频率。

  • 扩大窗口：移动右指针 right，不断将新字符加入窗口，并更新 window 中该字符的频率。如果该字符在 need 中，并且 window 中该字符的频率等于 need 中该字符的频率，说明该字符已经匹配完成，将 valid 计数器加 1。

  • 缩小窗口：当 valid 等于 needCount 时，说明当前窗口已经涵盖了 t 中的所有字符。此时，尝试移动左指针 left 缩小窗口，同时更新 window 中字符的频率。如果移除某个字符后，该字符在 window 中的频率小于 need 中该字符的频率，说明该字符不再匹配，将 valid 计数器减 1。

  • 记录最小子串：在每次找到满足条件的窗口时，记录当前窗口的长度和起始位置，更新最小子串的长度和起始位置

  • 代码
  • getOrDefault(键,默认值) 方法是 Map 接口（包括 HashMap）的一个常用方法，其设计目的是为了简化 “获取键对应的值，若不存在则返回默认值” 的操作。

8.力扣42.接雨水（单调栈、双指针、动态规划）

  • 思路

局部分析：

区间内能接的雨水由左边最高和右边最高决定，左右最高较矮的一个乘区间长度-区间内的方块=区间内能接的雨水

单调栈分析(横向填平)：

从低到高将栈顶元素填平

将栈顶元素用雨水填平，栈顶元素-1和枚举元素i中高度较小的来填平栈顶元素对应的横向面

双指针分析(单个元素)：

单个元素能接的雨水由左右最高元素决定，遍历过程分别记录左右最高元素，能接的雨水=左右最高中较小的高度-自身高度

左右指针l,r分别在两侧往中间靠拢

当左边最大小于右边最大时计算左指针的雨水 rain+=leftMax-height[l] 左指针右移

当右边最大小于左边最大时计算右指针的雨水 rain=rightMax-height[r] 右指针左移
