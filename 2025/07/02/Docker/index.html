3.命令解读

  • docker run：创建运行一个容器（-d 表示后台运行，-p 映射端口，-v 挂载卷）。

  • docker ps：查看正在运行的容器（-a 查看所有容器）。

  • docker start 容器名/ID：运行容器。

  • docker stop 容器名/ID：停止容器。

  • docker restart 容器名/ID：重启容器。

  • docker rm 容器名/ID：删除容器。

  • docker logs:查看容器的运行日志（-f:实时刷新尾部的日志）

  • docker images：查看本地镜像。

  • docker pull 镜像名/镜像名:版本：拉取远程镜像。

  • docker push 镜像名/镜像名:版本：推送镜像到仓库。

  • docker exec -it 容器名 bash:进入容器交互终端的命令

  • exit:退出容器终端返回宿主机

5.网络 -容器间通信

  • 容器之间通过Docker分配的IP地址实现通信。但是容器的IP地址是随机的，所以容器重新启动后要实现通信需要修改配置文件指定对应IP地址，操作繁琐。

  • 通过自定义网络之间根据容器的名字来实现通信

  • docker 容器默认三块网卡，创建的网络默认桥接模式

如果先创建容器后再加入网络，对应的容器会有两个网络，一个是创建时分配的默认bridge，一个时后面加入的网络
