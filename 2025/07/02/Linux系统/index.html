2. Windows 生成密钥对

在打开的终端中，执行以下命令来生成 SSH 密钥对，并使用邮箱作为注释信息：

  • -t rsa：指定密钥类型为 RSA，这是目前较为常用的 SSH 密钥类型。

  • -b 4096：设置密钥的位数为 4096，位数越高，密钥越安全。

  • -C "your_email@example.com"：为生成的密钥添加注释信息，通常使用你的邮箱地址，这样在管理多个密钥时可以方便识别。

  • 执行上述命令后，会有以下提示：

    如果你想使用默认路径和文件名，直接按下回车键即可。若要自定义保存路径和文件名，可以输入相应的路径和文件名。

    接着，会提示你设置一个密码：

    如果你希望在使用密钥时无需输入密码（即实现真正的免密登录），可以直接按下回车键；若要为密钥设置额外的安全保护，可输入一个强密码，然后再次确认密码。

    完成上述步骤后，SSH 密钥对就会生成。默认情况下，私钥文件名为 id_rsa，公钥文件名为 id_rsa.pub，它们位于 C:\Users\YourUsername\.ssh 目录下（YourUsername 是你的 Windows 用户名）。

3. Linux 保存 Windows 的公钥

将生成的公钥复制到你要连接的 Linux 系统的 ~/.ssh/authorized_keys 文件中，以实现免密登录。

2.linux命令基础

无论是什么命令，用于什么用途，在Linux中，命令有其通用的格式：

command [-options] [parameter]

command :命令本身

-option :可选，非必选，命令的一些选项，控制命令的行为细节

parameter :可选，非必选，命令的参数，多数用于命令的指向目标等

【】表示可选的意思

2.1 什么是命令、命令行

  • 命令：Linux操作指令，是系统内置的程序，可以以字符化的形式去使用
  • 命令行：即Linux终端，可以提供字符化的操作液面供命令执行

2.2Linux命令的通用格式

command [-options] [parameter]

  • 命令本体
  • 可选选项，控制命令的行为细节
  • 可选参数，控制命令的指向目标
  • 不使用如何命令默认路径在home下，/home/用户名，第一个/表示根目录下。实例：/home/xing

5.2复制、移动、删除（cp、mv、rm）

  1. cp命令复制文件文件夹

cp命令可以用于复制文件\文件夹，cp命令来自英文单词：copy

  • 语法：cp [-r] 参数1 参数2
  • -r选项：可选，用于复制文件夹使用，表示递归return
  • 参数1，Linux路径，表示被复制的文件或文件夹
  • 参数2，Linux路径，表示要复制去的地方

  1. mv移动文件或文件夹

mv命令可以用于移动文件、文件夹，来自单词：move

  • 语法：mv 参数1 参数2
  • 无选项
  • 参数1，Linux路径，表示被移动的文件或文件夹
  • 参数2，Linux路径，表示要移动去的目录，如果目标目录不存在，则进行重命名

  1. rm删除文件或文件夹

rm命令可以删除文件、文件夹，来自单词：remove

  • 语法：rm [-r -f] 参数1 参数2 ... 参数N
  • -r选项：和cp命令一样，-r选项用于删除文件夹
  • -f选项：表示force，强制删除（不会弹出提示确认消息）
    • 普通用户删除内容不会弹出提示，只有root管理员用户删除内容会有提示，所以一般用户用不到-f选项
  • 参数1、参数2…参数N表示要删除的文件或者文件夹路径，按照空格隔开

通配符

  • 符号*表示通配符，即匹配任意内容（包含空），示例：
    • test*,表示匹配任何以test开头的内容
    • *test，表示匹配任何以test结尾的内容
    • *test *，表示匹配任何包含test的内容

注意：

用mkdir创建的是文件夹（目录）rm或者rm -f都无法删除，必须用-r选项。

10.防火墙

在 Linux 系统中，常见的防火墙有 iptables、firewalld 和 ufw ，不同的发行版可能默认使用不同的防火墙工具，以下为你详细介绍各防火墙的操作：

1. iptables

iptables 是一个基于规则的防火墙工具，在许多 Linux 发行版中被广泛使用，尤其是一些较旧的系统。

查看规则

  • -L 表示列出当前的规则链。
  • -n 表示以数字形式显示 IP 地址和端口号，而不是解析为域名和服务名。
  • -v 表示显示详细信息。

添加规则

允许特定 IP 地址访问 SSH 服务（端口 22）：

  • -A 表示将规则添加到指定链的末尾。
  • -s 用于指定源 IP 地址。
  • -p 用于指定协议（如 tcp 或 udp）。
  • --dport 用于指定目标端口。
  • -j 用于指定动作（如 ACCEPT 表示允许，DROP 表示丢弃）。

删除规则

删除刚刚添加的允许特定 IP 访问 SSH 的规则：

  • -D 表示删除指定的规则。

1. 基本语法

  • **command**：要运行的命令或程序。
  • **[arguments]**：命令的参数。
  • **&**：可选，用于将命令放入后台执行（通常与nohup配合使用）。

后台运行命令并忽略挂断信号

  • > custom_output.log：将标准输出重定向到custom_output.log。
