1.linux的目录结构

1.1目录结构

linux的目录结构是一个树型结构

Windows系统可以拥有多个盘符，如C盘、D盘、E盘

linux没有盘符这个概念，只有一个根目录/，所有文件都在它下面

1.2Linux路径的描述方式

在Linux系统中，路径之间的层级关系，使用：/来表示

在Windows系统中，路径之间的层级关系，使用：\来表示

win: D:\data\work\hello.txt

Linux: /use/local/hello.txt

2.linux命令基础

无论是什么命令，用于什么用途，在Linux中，命令有其通用的格式：

command [-options] [parameter]

command :命令本身

-option :可选，非必选，命令的一些选项，控制命令的行为细节

parameter :可选，非必选，命令的参数，多数用于命令的指向目标等

【】表示可选的意思

2.1 什么是命令、命令行

  • 命令：Linux操作指令，是系统内置的程序，可以以字符化的形式去使用
  • 命令行：即Linux终端，可以提供字符化的操作液面供命令执行

2.2Linux命令的通用格式

command [-options] [parameter]

  • 命令本体
  • 可选选项，控制命令的行为细节
  • 可选参数，控制命令的指向目标
  • 不使用如何命令默认路径在home下，/home/用户名，第一个/表示根目录下。实例：/home/xing

3.常用基本命令

ls命令的作用是列出目录下的的内容，语法细节如下：

ls [-a -l -h] [linux路径]

  • -a -l -h 是可选的选项
  • Linux路径是此命令可选的参数

当不使用选项和参数，直接使用ls命令本体，表示：以平铺形式列出当前工作目录下的内容

Linux的命令行终端，在启动的时候，默认会加载：

  • 默认当前登录用户的home用户作为当前工作目录，所以ls命令列出的是home目录的内容
  • home目录：每个Linux操作用户在Linux系统的个人账户目录，路径在：/home/用户名
    • 如用户为xing，其home目录是：/home/xing (Windows也有)

3.1 ls命令的参数和选项（可缩写成l)

ls / :展示根目录下的内容

  • -a :表示all，列出全部文件包括隐藏文件（以.开头的文件表示的是Linux系统中的隐藏文件/文件夹）只有通过-a才可以看到
  • -l :以列表形式展示
  • -h :表示以易于阅读的形式（如K、M、G），列出文件的大小（必须搭配-l一起使用单独使用没有作用）

混合使用

3.2 -cd -pwd 命令

3.2.1 cd 切换工作目录（change directory)

语法：cd [linux路径]

  • cd命令无需选项，只有参数，表示要切换到那个目录下
  • cd命令直接执行，不写参数，表示回到用户的home目录

3.2.2 pwd 查看当前工作目录(Print Work Directory)

语法：pwd

  • pwd命令，无选项，无参数，直接输入pwd即可

特殊的路径表达符

  • . 表示当前目录，比如cd ./Desktop表示切换到当前目录下的Desktop目录内，和cd Desktop效果一样
  • .. 表示上一级目录，比如：cd .. 即可切换到上一级目录，cd ../..切换到上二级的目录
  • ~ 表示home目录，cd ~切换到home路径，和cd不加东西效果一样，cd ~/Desktop,切换到home内的Desktop目录

3.3-who命令

显示当前用户，无选项，无参数

5.文件操作命令

5.1创建、查看（touch、cat、more）

  1. touch`创建文件
  • 语法：touch Linux路径
  • touch命令无选项，参数必填，表示要创建的文件路径，相对、绝对、特殊路径符均可以使用

注意：区分文件和文件夹（目录），蓝色的为文件夹，白色的为文件,也可以通过查看目录下文件的详细信息来区分，ls -l

第一列信息表示权限信息，其中第一个字母为d表示为文件夹，-表示文件。

  1. cat查看文件内容
  • 准备好文件内容，可以通过cat查看文件内容
  • 语法：cat Linux路径
  • 无选项，只有必填参数，相对绝对特殊路径符都可以

  1. more命令查看文件内容

    • more也可以查看文件内容，和cat有略微不同
    • cat直接将内容全部显示出来
    • more支持翻页，如果文件内容过多，可以一页页的展示
    • 语法：more Linux路径
    • 无选项，只有必填参数，相对绝对特殊路径符都可以

    Linux系统内置一个文件，路径为：/etc/services,可以通过more查看

    • 查看过程中通过空格翻页
    • 通过q退出查看

5.3复制、移动、删除（cp、mv、rm）

  1. cp命令复制文件文件夹

cp命令可以用于复制文件\文件夹，cp命令来自英文单词：copy

  • 语法：cp [-r] 参数1 参数2
  • -r选项：可选，用于复制文件夹使用，表示递归return
  • 参数1，Linux路径，表示被复制的文件或文件夹
  • 参数2，Linux路径，表示要复制去的地方

  1. mv移动文件或文件夹

mv命令可以用于移动文件、文件夹，来自单词：move

  • 语法：mv 参数1 参数2
  • 无选项
  • 参数1，Linux路径，表示被移动的文件或文件夹
  • 参数2，Linux路径，表示要移动去的地方，如果目标不存在，则进行改名，确保目标存在·

  1. rm删除文件或文件夹

rm命令可以删除文件、文件夹，来自单词：remove

  • 语法：rm [-r -f] 参数1 参数2 ... 参数N
  • -r选项：和cp命令一样，-r选项用于删除文件夹
  • -f选项：表示force，强制删除（不会弹出提示确认消息）
    • 普通用户删除内容不会弹出提示，只有root管理员用户删除内容会有提示，所以一般用户用不到-f选项
  • 参数1、参数2…参数N表示要删除的文件或者文件夹路径，按照空格隔开

通配符

  • 符号*表示通配符，即匹配任意内容（包含空），示例：
    • test*,表示匹配任何以test开头的内容
    • *test，表示匹配任何以test结尾的内容
    • *test *，表示匹配任何包含test的内容

注意：

用mkdir创建的是文件夹（目录）rm或者rm -f都无法删除，必须用-r选项。

演示强制删除，-f选项

  • 可以通过sudo -i,并输入密码，进入超级管理员用户
  • 输入exit命令，退回普通用户

### shell变量

- 定义变量时，变量名不加美元符号$(php语言中变量需要)

- ~~~shell
  your_name="zhuzhuxia"
  

- 除了显示的直接赋值，还可以用语句给变量赋值

- ~~~shell
  for file in $(ls /etc）   #将/etc下目录的文件名循环出来
  

- ### 只读变量

- 使用readonly命令可以将变量定义为只读变量，只读变量的值不能被改变

- ~~~shell
  #！/bin/b
  my_name="zhuzhuxia"
  readonly my_name
  my_name="chaorenqiang"
  

- unset不能删除只读变量

- ### 不同类型的变量

- ~~~bash
  #字符串
  my_string="Hellow,world!"#在shell中变量通常被视为字符串，你可以使用单引号和双引号来定义
  #整数变量  可以使用declare或typeset命令来声明整数变量
  declare -i my_integer=42

  declare -A associative_array

  greeting_3='hello, ${your_name} !'
  echo $greeting_2  $greeting_3
  
  #hello, runoob ! hello, runoob !
  #hello, runoob ! hello, ${your_name} !
